# 🚀 ConcreteBill - Development Setup Guide

## Quick Start

```bash
# 1. Clone and install
git clone https://github.com/smith-ch/croncretebill.git
cd croncretebill
pnpm install

# 2. Setup environment
cp .env.example .env.local
# Edit .env.local with your Supabase credentials

# 3. Start development
pnpm dev
```

## 🛠️ Development Scripts

| Command | Description |
|---------|-------------|
| `pnpm dev` | Start development server |
| `pnpm build` | Build for production |
| `pnpm start` | Start production server |
| `pnpm lint` | Run ESLint |
| `pnpm verify-pwa` | Verify PWA configuration |

## 📦 Key Dependencies

### Production
- **Next.js 14.2.16** - React framework
- **TypeScript 5.0** - Static typing
- **Tailwind CSS 3** - Styling
- **Supabase** - Backend & Database
- **Radix UI** - Component primitives
- **next-pwa 5.6.0** - PWA functionality

### Development
- **ESLint** - Code linting
- **TypeScript** - Type checking
- **Prettier** - Code formatting

## 🌐 Supabase Setup

1. Create new project at [supabase.com](https://supabase.com)
2. Run database scripts in order:
   ```bash
   # Execute in Supabase SQL Editor
   scripts/01-create-tables.sql
   scripts/02-seed-data.sql
   # ... continue with all numbered scripts
   ```
3. Configure RLS policies
4. Setup storage buckets

## 📱 PWA Development

1. **HTTPS Required**: Use `https://localhost:3000` for PWA testing
2. **Service Worker**: Auto-generated by next-pwa
3. **Manifest**: Located at `/public/manifest.json`
4. **Icons**: Generated from `/public/concretebill-icon.svg`

## 🐛 Common Issues

### PWA Not Installing
- Check HTTPS is enabled
- Verify manifest.json is accessible
- Confirm service worker is registered
- Use `/pwa-status` page for diagnosis

### Database Connection
- Verify Supabase URL/keys in .env.local
- Check RLS policies are enabled
- Confirm database scripts ran successfully

### Build Errors
- Clear `.next` folder: `rm -rf .next`
- Reinstall dependencies: `pnpm install`
- Check TypeScript errors: `pnpm type-check`
